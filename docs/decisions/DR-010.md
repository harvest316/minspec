# DR-010: ScroogeLLM telemetry decision — relocated

> **Tombstone.** This decision concerned ScroogeLLM's opt-in telemetry. Its full
> content was moved to the private `harvest316/scroogellm` repository per
> [DR-027](DR-027.md) — ScroogeLLM is not public. This stub preserves the register
> slot so `DR-010` references (from [DR-013](DR-013.md), [DR-015](DR-015.md), and
> [EPIC-006](../epics/EPIC-006-trust-and-supply-chain.md)) resolve.
>
> The decision itself lives in the private repo. See [DR-027](DR-027.md).

> **Scope note for the sections below.** The original telemetry decision is NOT
> reproduced here (it is private, per [DR-027](DR-027.md)). The cross-check sections

> **leave a tombstone stub in the public register rather than delete the `DR-010`
> slot.** They do not re-litigate ScroogeLLM telemetry.

## Risks & Mitigations

| Risk | Mechanism / anchor | Mitigation |
|---|---|---|
| Inbound references silently break if the slot is deleted instead of stubbed | [DR-013](DR-013.md), [DR-015](DR-015.md), and [EPIC-006](../epics/EPIC-006-trust-and-supply-chain.md) all cite `DR-010` by number; a gap would dangle | This tombstone keeps the slot occupied so every `DR-010` link resolves to an explanation |
| Reader assumes the telemetry decision was reversed/abandoned, not relocated | `status: accepted` in the frontmatter with no decision body is ambiguous — "accepted but empty" reads as withdrawn unless the move is spelled out | Tombstone states explicitly the content *moved* to `harvest316/scroogellm` and points at [DR-027](DR-027.md) §4 (tombstones replace moved entries) as the why |
| Slot number gets reused by the local sequential allocator | Local register allocates `max(DR-NNN)+1`; a deleted file could let `DR-010` be re-minted for an unrelated decision, corrupting the [DR-013](DR-013.md)/[DR-015](DR-015.md) back-references | Keeping the file reserves `DR-010` permanently against reuse |
| Private content leaks back into the public stub on a careless edit | The relocated telemetry text must stay out of public `~/code/MinSpecPro` (see project memory: scrooge privatization) | The stub carries no decision substance to leak; the scope note above forbids re-importing it |

## Consequences

**Positive:**
- Every existing `DR-010` reference — from [DR-013](DR-013.md), [DR-015](DR-015.md), [EPIC-006](../epics/EPIC-006-trust-and-supply-chain.md) — still resolves to a page that explains where the decision went.
- The public register stays consistent with [DR-027](DR-027.md)'s rule that ScroogeLLM material is private, without leaving a numbering hole.
- `DR-010` is permanently reserved against re-minting by the local `max(DR-NNN)+1` allocator.

**Negative:**
- The decision's rationale is now split across two repos: a reader must follow the link into private `harvest316/scroogellm` to see the actual telemetry choice, so the public register alone is no longer self-contained for `DR-010`.
- A dead/duplicated slot persists in the register forever (the cost of never reusing the number).

## Alternatives Considered

- **Delete `DR-010` entirely.** Rejected: it would dangle the inbound references from [DR-013](DR-013.md), [DR-015](DR-015.md), and [EPIC-006](../epics/EPIC-006-trust-and-supply-chain.md), and free the number for accidental reuse by the local allocator.
- **Keep the full telemetry text public and merely cross-link.** Rejected: violates [DR-027](DR-027.md)'s privatization of all ScroogeLLM material — the content must not live in public `~/code/MinSpecPro`.
- **Renumber the inbound references to point at the private DR instead of `DR-010`.** Rejected: public DRs/epics cannot cite a private-repo decision number meaningfully, and it would churn three already-published documents.

## Costly to Refactor

Cheap to reverse — ADR filter: yes, undoable in well under a day. The stub holds no decision substance; reverting would mean deleting this file or re-expanding the body, a single edit. The only thing genuinely costly to undo is the *relocation* itself (owned by [DR-027](DR-027.md), not this stub) — re-publishing ScroogeLLM's telemetry decision would reopen the privatization decision, which is out of scope here.
