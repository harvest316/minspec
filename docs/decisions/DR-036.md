# DR-036: Autopilot Mode — single-gate autonomous build pipeline

## Context

MinSpec's HITL model requires human approval at every spec, plan, task-list, and PR.
For throwaway "playground" repos (MeetLoop, HireLoop, SourceBridge) where a wrong
build costs nothing real, the cost/friction of per-artifact approval is higher than the
risk. A compressed alternative — one human gate, then fully autonomous — is worth
trialling.

Three design takes were synthesised into a single design doc
(`docs/autopilot-mode-design.md`), then hardened against three independent
code-level adversarial reviews. Twelve verified attack surfaces (A1–A12) were
identified; each has a required mitigation. The design was submitted for PO greenlight.

## Decision

**Greenlight Autopilot Mode for SourceBridge trial**, subject to the §7 open-decision
defaults and the §4 must-fix prerequisites:

### Open decisions (all accepting the recommended option)

| Decision | Choice |
|---|---|
| Approval-store integrity | Commit `approvals.json` for playground repos (reverse gitignore) |
| Backstop scope | Build A1–A5 + A11 first; A6–A10 gate on legal-invariant requirements only |
| Prototype fidelity | Deterministic skeleton first, styled later |
| Refuter tie-breaker model | Cross-tier (Opus), not same-model monoculture |
| First-trial repo | SourceBridge |

### What Autopilot Mode does

The human approves **one composite artifact (the Blueprint)** — an overview brief +
a navigable clickable prototype — via `MinSpec: Approve Blueprint`. That single
approval hash-locks both halves, marks child specs `blueprint`-derived, and releases
the autonomous spine:

`Blueprint gate → Spec fan-out → Plan → Tasks → Implement + deploy (auto-merge)`

Each phase is **gate → adversarial-refute → advance**. Any phase that fails in 3
bounded retries, or whose refuters surface an unresolved defect, writes
`status: human-needed` and surfaces in the PR-review-queue pane (DR-033).

### What it does NOT do

- Never touches MinSpec core — guarded by git-identity re-verification each iteration (A11)
- Never auto-builds `human-only` requirements (legal, financial, irreversible, marketing)
- Never emits to a real outbound channel without a human arm-token (A8)
- Never accepts a bare LLM `CLEAN` as a gate-pass — must be backed by a
  machine-checkable artifact (A9)

## Must-fix prerequisites (A1–A5 + A11) — required before first run

| Attack | Fix required |
|---|---|
| A1 (gate guards empty set) | Deliverable registry — every generated output path declared in its owning spec |
| A2 (self-asserted status) | External pipeline-state file the agent cannot write, OR status-agnostic gating |
| A3 (forged approvals) | `approvedBy: human\|blueprint` + `derivedFrom: <blueprintHash>` on every record; commit `approvals.json`; PreToolUse deny on `.minspec/config.json` + role prompts |
| A4 (hash doesn't bind prototype) | Inline prototype content digest as text inside the overview bytes |
| A5 (symmetric primitive never errors) | Per-repo severity-elevation seam; T0 test proving missing-tier/epic/acceptance → `complete:false` |
| A11 (cross-repo scope leak) | Re-derive auto-merge from verified git identity each loop iteration; hard-refuse non-playground repos |
