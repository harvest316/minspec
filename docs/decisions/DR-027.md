# DR-027: ScroogeLLM is private; MinSpec monorepo stays public

## Context

The `harvest316/minspec` monorepo is **public**. It was framed as a two-extension
monorepo (MinSpec + ScroogeLLM) plus an extension pack. MinSpec is the open, free,
Tier-0 SDD tool — public by design ([DR-004](DR-004.md)). ScroogeLLM is the freemium
proxy whose **defensible IP — the proxy-layer logic and the measurement methodology
behind it — cannot be public.** That extends beyond code: the design specs and
competitive research describe the approach in enough detail to be copied.

The public repo also gave visitors no signal that nothing is shipped yet.

## Decision

1. **MinSpec monorepo stays PUBLIC.** It hosts MinSpec (open), `packages/shared`
   (classifier engine), and the `packages/extension-pack` manifest.
2. **Everything ScroogeLLM moves to a separate PRIVATE repo**
   (`harvest316/scroogellm`): source, the spec (`specs/scroogellm/`), `EPIC-008`,
   `DR-007` (plugin arch), `DR-010` (telemetry), and the scrooge competitive/legal
   research under `docs/research/`. The extension pack references
   `aiclarity.scroogellm` **by marketplace ID**, so it is location-agnostic.
3. **Removed the `packages/scroogellm/` scaffold stub** from the public repo (it was
   harmless — `package.json`/`LICENSE`/`tsconfig`, zero source — but implied the
   source belonged here). Not a build dependency; build unaffected.
4. **Tombstones** replace the moved register entries (`DR-007`, `DR-010`, `EPIC-008`)
   so the sequential register stays gap-free and numeric cross-references resolve.
5. **Public git history was scrubbed** (`git filter-repo`, force-push) to purge the
   moved files' historical blobs. Mitigation, not guarantee: the docs were public
   for ~1 week, so assume some external copy may exist.
6. **Public repo is honestly labelled "under construction"** — README banner +
   shields badges + repo About/topics.

## Alternatives considered

- **Whole monorepo private.** Simplest, zero leak surface — but kills MinSpec's
  open/public positioning and makes a public WIP signpost moot. Rejected: MinSpec is
  meant to be open.
- **Private scrooge as a git submodule inside the public monorepo.** Keeps the slot;
  code stays private. Rejected: leaks existence/name, adds submodule complexity for a
  solo dev, for no gain over a clean separate repo.
- **Leave docs public / redact in place.** Rejected: the moat is described as an
  implementable method in the design + research; partial redaction is error-prone and
  the docs track a private product anyway.

## Consequences

**Positive:**

- ScroogeLLM's defensible IP (proxy-layer logic + measurement methodology) and its
  competitive/legal research under `docs/research/` are no longer in a public repo —
  the moat described in the Context is removed from public view.
- `DR-007` / `DR-010` / `EPIC-008` survive as **tombstones** pointing to the private
  repo, so the sequential register stays gap-free and prose references in `DR-013` /
  `DR-015` still resolve — numeric cross-references don't dangle.
- The extension pack still references `aiclarity.scroogellm` **by marketplace ID**
  (Decision item 2), so the pack manifest is location-agnostic and unaffected by the
  move.
- Removing the `packages/scroogellm/` scaffold stub (Decision item 3) drops the false
  signal that scrooge source belonged in the public repo; build is unaffected (it was
  not a build dependency).

**Negative:**

- The "two-extension monorepo" framing is retired; this repo is now **MinSpec +
  shared + pack manifest**. README, package table, and license table all had to be
  updated to match.
- History rewrite (`git filter-repo`, force-push, Decision item 5) changed every commit
  SHA on `main`. Any other clone must `git fetch && git reset --hard origin/main` (or
  re-clone); a pre-scrub backup bundle is retained locally as the only escape hatch.
- The two merged agent branches that carried scrooge content (`agent/issue-1`,
  `agent/issue-21`) had to be deleted from origin (the orphan `badges` branch, no
  scrooge, was left intact) — branch references that pointed at them are now broken.
- The scrub is mitigation, not guarantee: the docs were public for ~1 week, so an
  external copy may already exist (Decision item 5).
- Interacts with [DR-015](DR-015.md) (packaging of the separate Execute extension) and
  [DR-018](DR-018.md) (per-package licensing — the MIT row no longer lists scroogellm),
  so both must be kept consistent with the new repo boundary.

## Risks & Mitigations

| Risk | Mechanism / anchor | Mitigation |
|---|---|---|
| The moat already leaked during the ~1-week public window | Decision item 5 — docs were public before the `git filter-repo` scrub; force-push only purges the blobs from `harvest316/minspec`, not forks/clones/caches/archives | Audit tracked as **#120**; treat the methodology as potentially-known and keep the *measurement* moat (per memory: secret sauce is route/cache/measure, not text) evolving in the private repo |
| A stale clone resurrects the scrubbed blobs via push | History rewrite changed every SHA on `main` (Consequences → Negative); a clone that still has pre-scrub objects could re-introduce them | Documented recovery (`git fetch && git reset --hard origin/main`); pre-scrub backup bundle kept *locally* only, not pushed |
| Numeric cross-references to moved entries dangle | `DR-007` / `DR-010` / `EPIC-008` were physically moved out (Decision item 2); `DR-013` / `DR-015` reference them in prose | Tombstones (Decision item 4) left in place pointing to `harvest316/scroogellm`, keeping the register gap-free |
| Pack build breaks because scrooge is no longer in-tree | `packages/extension-pack/package.json` lists scrooge in its `extensionPack` array; if that entry were a workspace path it would now dangle | The entry is the **marketplace ID** `aiclarity.scroogellm` (Decision item 2), not a workspace path, so it resolves regardless of repo; removing the `packages/scroogellm/` stub left the build green (Decision item 3) |
| License table still implies scrooge is MIT in this repo | [DR-018](DR-018.md) per-package licensing; scrooge is proprietary, not MIT (per memory: legal entity Audit&Fix) | MIT row updated to drop scroogellm (Consequences); DR-018 must stay consistent with the new boundary |

## Costly to Refactor

Moderate-to-high blast radius, and **not** undoable in <1 day — fails the ADR filter,
which is why this is a DR. The repo split, the `git filter-repo` history rewrite, and
the force-push are effectively one-way: every `main` SHA changed, the moved files
(`specs/scroogellm/`, `DR-007`, `DR-010`, `EPIC-008`, `docs/research/`) now live in
`harvest316/scroogellm`, and any contributor clone must hard-reset or re-clone. Merging
scrooge *back* into the public monorepo would re-expose exactly the IP this decision
removed, so the privacy half is intentionally hard to reverse. The cheap-to-reverse
parts are cosmetic only: the "under construction" banner/badges (Decision item 6) and
the README/package/license table edits can be undone in minutes. Reversing the *move*
itself (re-homing the private docs, re-publishing history) is the costly part and would
itself need a new DR.
