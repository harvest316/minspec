## Context

The tier classifier ([classifier.ts](../../packages/minspec/src/lib/classifier.ts))
and its analyzers ([git-analyzer.ts](../../packages/minspec/src/lib/git-analyzer.ts),
ast-analyzer.ts) are validated only by synthetic unit tests. We have no evidence
the tier thresholds (file count, line count, file-type diversity) produce sensible
tiers on real-world code changes.

We want to validate against a real corpus of issue→PR pairs. **SWE-bench-Verified**
(500 human-vetted GitHub issue→gold-patch instances) is the best fit: each instance
has a unified-diff patch and a problem statement.

Two constraints collide with this:

1. **Invariant #2 (tiered network consent / DR-004):** no `http`/`https`/`fetch` in
   `packages/minspec` or `packages/shared`. The extension and its tests must stay
   offline. SWE-bench data lives on the network.
2. **Repo hygiene:** SWE-bench patches are large and carry upstream licensing. We do
   not want them in git history.

## Decision

Validate via an **out-of-tree fixture** model:

- A **standalone fetch script** (not part of the extension build, not imported by any
  extension code) downloads a curated ~50-instance subset of SWE-bench-Verified into
  a **gitignored** directory. This script is the *only* component permitted network
  access — consistent with DR-004 Tier 1 (delegated, out of the extension).
- **Patches are gitignored; labels are committed.** Hand-assigned `expectedTier`
  labels, keyed by `instanceId`, live in a committed file. The data they reference is
  fetched locally and never committed.
- The harness **reuses the real path**: apply each patch to a temp git repo, stage,
  call the real `analyzeGitDiff()` → `classify()`. No analyzer logic is
  reimplemented in test code.
- The harness **skips, not fails**, when the gitignored data dir is absent — so
  `npm test` is green on a fresh, offline clone (invariant #2 + developer ergonomics).
- The harness **measures only**. It reports accuracy + a confusion matrix; it does
  not mutate classifier weights. Weight tuning remains the separate calibration path
  already in classifier.ts.

## Consequences

### Positive
- Real-diff evidence for tier thresholds, with a concrete accuracy metric.
- Invariant #2 preserved: zero network reaches the extension or its committed tests.
- Fresh clones and network-less CI stay green (graceful skip).
- Real code path is exercised — the harness can't pass while the shipped analyzer is wrong.

### Negative
- Hand-labelling ~50 instances is manual effort (one-time, plus top-ups).
- Contributors must run the fetch script before they can reproduce the accuracy run.
- Temp-repo-per-instance is slower than parsing patches directly — accepted for
  fidelity (we test the real analyzer, not a patch parser).

### Risks accepted
- Hand labels are subjective. Mitigated by a written rubric in design.md and by
  treating outliers (large expected/predicted gaps) as the primary review signal,
  not the headline accuracy number alone.
