## Context

The tier classifier ([classifier.ts](../../packages/minspec/src/lib/classifier.ts))
and its analyzers ([git-analyzer.ts](../../packages/minspec/src/lib/git-analyzer.ts),
ast-analyzer.ts) are validated only by synthetic unit tests. We have no evidence
the tier thresholds (file count, line count, file-type diversity) produce sensible
tiers on real-world code changes.

We want to validate against a real corpus of issue→PR pairs. **SWE-bench-Verified**
(500 human-vetted GitHub issue→gold-patch instances) is the best fit: each instance
has a unified-diff patch and a problem statement.

Two constraints collide with this:

1. **Invariant #2 (tiered network consent / DR-004):** no `http`/`https`/`fetch` in
   `packages/minspec` or `packages/shared`. The extension and its tests must stay
   offline. SWE-bench data lives on the network.
2. **Repo hygiene:** SWE-bench patches are large and carry upstream licensing. We do
   not want them in git history.

## Decision

Validate via an **out-of-tree fixture** model:

- A **standalone fetch script** (not part of the extension build, not imported by any
  extension code) downloads a curated ~50-instance subset of SWE-bench-Verified into
  a **gitignored** directory. This script is the *only* component permitted network
  access — consistent with DR-004 Tier 1 (delegated, out of the extension).
- **Patches are gitignored; labels are committed.** Hand-assigned `expectedTier`
  labels, keyed by `instanceId`, live in a committed file. The data they reference is
  fetched locally and never committed.
- The harness **reuses the real path**: apply each patch to a temp git repo, stage,
  call the real `analyzeGitDiff()` → `classify()`. No analyzer logic is
  reimplemented in test code.
- The harness **skips, not fails**, when the gitignored data dir is absent — so
  `npm test` is green on a fresh, offline clone (invariant #2 + developer ergonomics).
- The harness **measures only**. It reports accuracy + a confusion matrix; it does
  not mutate classifier weights. Weight tuning remains the separate calibration path
  already in classifier.ts.

## Costly to Refactor

- **The out-of-tree boundary** — network lives ONLY in the standalone fetch script (patches gitignored, labels committed). A leak of network into the extension/harness breaks DR-004 Tier-0. *Check: no network outside the fetch script.*
- Otherwise a harness — low ongoing refactor cost.

## Consequences

**Positive:**
- Real-diff evidence for tier thresholds, with a concrete accuracy metric — replaces
  the synthetic-only unit tests on `classifier.ts` flagged in Context.
- Invariant #2 preserved: zero network reaches the extension or its committed tests
  (network is confined to the standalone out-of-tree fetch script per DR-004 Tier 1).
- Fresh clones and network-less CI stay green — the harness skips, not fails, when the
  gitignored data dir is absent.
- Real code path is exercised: the harness calls the shipped `analyzeGitDiff()` →
  `classify()` on a temp git repo, so it can't pass while the shipped analyzer is wrong
  (no analyzer logic reimplemented in test code).

**Negative:**
- Hand-labelling ~50 SWE-bench-Verified instances (`expectedTier` keyed by `instanceId`)
  is manual effort — one-time, plus top-ups when the subset grows.
- Contributors must run the fetch script before they can reproduce the accuracy run; the
  default `npm test` only proves the harness skips cleanly, not that accuracy holds.
- Temp-repo-per-instance (apply patch → stage → analyze) is slower than parsing patches
  directly — accepted for fidelity (we test the real analyzer, not a patch parser).

## Risks & Mitigations

| Risk | Mechanism / anchor | Mitigation |
|---|---|---|
| Hand labels are subjective, so the accuracy number reflects labeller bias rather than classifier quality. | `expectedTier` is hand-assigned per `instanceId`. | Written rubric in `specs/minspec/classifier-validation/design.md`; treat outliers (large expected/predicted gaps) as the primary review signal, not the headline accuracy number alone. |
| Network leaks from the standalone fetch script into the extension/harness, breaking invariant #2 / DR-004 Tier-0. | Any `http`/`https`/`fetch` reaching `packages/minspec` or `packages/shared`. | The "no network outside the fetch script" check in the Costly-to-Refactor section; fetch script is not imported by extension code and patches stay gitignored. |
| Validation finding gets misread as "tune the thresholds" rather than "the classifier measures diff size, not difficulty" (the known under-tiering of subtle bugs). | Harness **measures only** — it reports accuracy + confusion matrix and does not mutate classifier weights. | Weight tuning stays on the separate calibration path already in `classifier.ts`; this DR's harness is explicitly non-mutating, keeping the measurement and the (open) tuning direction decoupled. |

## Alternatives Considered

- **Commit the SWE-bench patches into the repo (no gitignore).** Rejected: SWE-bench
  patches are large and carry upstream licensing (Context constraint #2) — they would
  bloat git history and import third-party license obligations. Committing only the
  `expectedTier` labels keyed by `instanceId` avoids both.
- **Fetch SWE-bench data from inside the extension/test code.** Rejected: this puts
  `http`/`https`/`fetch` into `packages/minspec`/`packages/shared`, directly violating
  invariant #2 and DR-004 Tier-0. The out-of-tree standalone fetch script is the only
  DR-004-consistent (Tier 1, delegated) place for network.
- **Parse the unified-diff patches directly in the harness instead of applying them to a
  temp git repo.** Rejected: a patch parser would be a second implementation of the
  analyzer's input path, so the harness could pass while the shipped `analyzeGitDiff()`
  is wrong. Applying to a real temp repo and calling the real path is the whole point;
  the speed cost is accepted (see Consequences → Negative).
- **Fail `npm test` when the gitignored data dir is absent.** Rejected: that breaks fresh
  offline clones and network-less CI (invariant #2 + developer ergonomics). The harness
  skips instead.
