# DR-033: Auto-triage + auto-build most raised issues

> Triggered by: session request 2026-06-05 (Paul Harvey) — *"work towards a way to
> auto-triage and auto-build most issues. if i asked for the issue to be raised, i think
> it's worth doing. and the priority you do them doesn't particularly matter to me. so
> long we stick to just-enough-human."*

## Context

The **global parking-lot rule** — *mmo-platform DR-360* (the **parent** register, ~DR-360;
**not** a decision in this repo's local register) — routes topic drift to GitHub issues
and states *"do NOT auto-start inbox issues; the user triages to a priority before any
agent works them."* This repo inherits that rule via the global CLAUDE.md.

Its consequence here: a ~90-issue backlog, **zero** `agent-ready`, every issue gated on
the human to triage before anything is built. Throughput is bottlenecked on a human step
the user does not value — they have stated the act of *raising* an issue is itself the
"worth doing" signal, and that priority ordering is not a judgment they want to own.

**Register-hygiene note (the trigger's sibling finding).** This project keeps its **own**
local register (currently to DR-032). `DR-360` and `DR-355` (escalation) are **parent /
global** numbers that do not resolve in this repo — citing them bare reads as a dangling
local ref (the inward case of the DR-032 / SPEC-021 cross-register-collision class). **This
DR is the local anchor** for the parking-lot + auto-build policy; future minspec artifacts
reference `DR-033`, and any citation of the global rule is written `mmo-platform DR-360`,
never bare.

## Decision

1. **Auto-build filter (broader-auto).** Issues labelled `bug` / `feat` / `chore` /
   `documentation` / `enhancement` / CI / gate-repairs **auto-build**. Held for the human:
   `idea` / marketing / positioning / copy, legal, `decide:` / monetization,
   irreversible-architecture, cross-product-schema / contract-migration, billing, and
   published-site content. The test per issue: does "done" = a **signal** the human can
   verify (tests green + PR matches the ask) or a **judgment** only the human can supply
   (taste, strategy, risk, irreversibility)? First → auto; second → human.

2. **Scope = all raised, including Claude-auto-parked.** Most inbox issues were parked by
   the agent under the global rule, not personally requested. A raised issue counts as
   worth doing regardless of who filed it — minus the held-for-human classes above.

3. **Human gate = intent + signal, never the diff — placed by blast radius
   (consequence-hybrid).** The human verifies INTENT at the plan/spec (the doc they
   actually read) and a SIGNAL at the output; they never review code line-by-line. *Where*
   the output gate sits is keyed to blast radius (reusing the consequence analyzers, #88):
   - **Low-blast** changes (isolated, reversible, covered by a regression that provably
     fails on the pre-fix code) **auto-merge** once their signals are green.
   - **High-blast** changes (exported/public surface, schema, irreversible, security-sink)
     **hold for a ~30s PR-summary read** before merge.
   The SDD **tier dictates ceremony** (how much spec gets written), not an extra approval
   gate. This holds **just-enough-human** by *moving* the gate, not removing it.

   **Sequencing (auto-merge has prerequisites).** Auto-merge over-trusts the agent's own
   tests unless two backstops exist: a PR self-report proving the regression **fails on old
   code** (#180), and a gate rejecting **hollow/tautological tests** (#130). Until both
   ship — plus blast-radius routing (#88) — the loop runs **PR-gate for everything** (every
   PR holds for the summary read). Low-blast auto-merge switches on only after the backstops
   land. *Where* the gate sits is itself a **per-dev/per-team setting** to surface to users
   (#183).

   **Default decision (amended 2026-06-06).** Of the three modes in #183, the recorded
   **default = consequence-hybrid** (option 3 above) — per-dev/per-team overridable, not
   hardcoded. Rejected as default: *plan-gate + auto-merge* (option 2), which auto-merges
   **high-blast** changes (schema, exported surface, irreversible, security-sink) unseen and
   has a **T1 hole** — a one-sentence-spec issue has no real plan to gate, so it would get
   zero human touch. Hybrid keeps the same low-friction flow for the boring 90% while the
   rare high-blast PR still gets a ~30s read. **Caveat (load-bearing dependency):** hybrid's
   safety rests entirely on the #88 blast-radius classifier routing correctly — a high→low
   misclassification auto-merges a change the human wanted to see. If #88 ships weak, hybrid
   degrades **silently** toward auto-merge-everything; the classifier's accuracy is therefore
   a release gate for enabling auto-merge, not an optional nicety. (Note: today, pre-backstop,
   all three modes behave identically — PR-gate-for-all — so this default only takes effect
   once #180 + #130 + #88 land.)

4. **Mechanics.** One **isolated git worktree** per issue (never branch-switch the shared
   checkout); RCDD for bugs / SDD ceremony per tier for feats (broader-auto: infer loose
   scope and **state assumptions in the PR body**); full `test + lint + build + validate`
   gate; **one PR per issue** (`Closes #N`); genuine blockers escalate to the human
   (`agent-escalated`) rather than ship a stub. Run via **`/loop`** — the dev-time
   **prototype of the agent-executor vsix** (`aiclarity.agent-execute`, EPIC-007).

5. **Amends the inherited default.** For this repo, this supersedes *mmo-platform DR-360*'s
   "do NOT auto-start inbox issues" — the auto-set defined above no longer waits on human
   triage. The global rule's *capture* mechanism (drift → issue) is unchanged.

## Risks & Mitigations

- **PR flood / shallow review.** → one PR per issue, self-paced reviewable cadence, user
  can pause the loop at any time. Nothing merges without the human.
- **Auto-building something that needed judgment.** → the filter; when an issue's "done"
  is ambiguous between signal and judgment, **escalate to the human, do not build**.
- **Concurrent-session collisions** (multiple live sessions on one checkout). → isolated
  worktrees only; skip issues with an open PR / `agent-running` / in-flight elsewhere;
  verify the shared checkout is untouched after each build.
- **The merge gate is load-bearing.** Auto-build produces PRs, not merges — the human
  review is the backstop that keeps a wrong auto-build out of `main`.
- **Blast-radius misclassification (hybrid default).** A high-blast change scored low
  auto-merges unseen. → #88 classifier accuracy is a **release gate** for turning auto-merge
  on; until measured, the loop stays PR-gate-for-all. A weak classifier degrades hybrid
  silently — surface its routing decision in the PR body so a misroute is visible.

## Costly to Refactor

- Low. The policy only changes *which issues start without a human triage step*; it is
  reversible by re-imposing the triage gate. The mechanics (worktree-per-issue, one PR)
  are already the repo's norm.

## Consequences

- The backlog drains without per-issue human triage; human time concentrates at PR review.
- The local register gains the canonical anchor for parking-lot + auto-build; parent-register
  citations (`DR-360`, `DR-355`) are now attributable and a cleanup of the bare leaks is
  tracked.
- The dev-time loop's filter + gates inform the shipped agent-execute product surface (#173).
- **PR review is now a first-class human queue.** With the gate at PR approval, unreviewed
  PRs ARE the human's work — they must be **surfaced**, not inferred. Two surfaces: the
  next-human-task signpost must rank "review PR #N" alongside spec/plan approvals (#182),
  and a MinSpec explorer pane must show the full PR-review queue (#211). Without these the
  signpost would say "nothing to do" while PRs sit unreviewed — a never-wrong-signpost
  violation (SPEC-012 single-deterministic-next-task contract). Both are conditional on
  gate = PR (no-op under plan-approval + auto-merge).

## Alternatives considered

- **Keep the human triage gate (status quo).** Rejected: bottleneck, and "raised ⇒ worth"
  makes a second human triage redundant ceremony.
- **One-shot parallel backlog sweep.** Rejected: a flood of simultaneous PRs is less
  reviewable than a paced one-PR-per-issue loop.
- **Auto-build everything, including strategic/marketing/legal.** Rejected: those need
  supplied judgment, not signal verification — they stay with the human (the filter).
