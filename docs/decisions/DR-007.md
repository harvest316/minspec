> [DR-027](DR-027.md) — ScroogeLLM is not public. This stub preserves the register

> public/private split rationale.

## Risks & Mitigations

The decision recorded *here* is narrow — keep this slot as a tombstone rather than
deleting `DR-007` outright. Its risks are register-integrity risks, not architecture
risks (the architecture moved to the private repo per [DR-027](DR-027.md) Decision
item 2):

| Risk | Mechanism / anchor | Mitigation |
|---|---|---|
| A reader treats this stub as the live plugin-architecture decision and acts on stale assumptions | The `title:` frontmatter still says "ScroogeLLM plugin architecture" | The tombstone blockquote states up front the content moved and points to [DR-027](DR-027.md); `relates_to: [DR-027, EPIC-008]` makes the redirect machine-readable |
| The two copies (public tombstone + private real DR) drift, or the private `DR-007` is renumbered | [DR-027](DR-027.md) Decision item 2 lists `DR-007` (plugin arch) among the entries moved to `harvest316/scroogellm`, but does not pin its number — the no-renumber rule lives in the register convention, not DR-027 | The project CLAUDE.md Decision Register rule "It does NOT share the global register … keeps its OWN local register" plus the imported-DR clause (DR-007/010 "imported there with their original numbers", per CLAUDE.md SDD-Phases note) forbids renumbering imported DRs — both repos hold `DR-007` |
| Deleting the stub instead of tombstoning would gap the sequence and break numeric cross-refs | The only inbound references to `DR-007` are [DR-027](DR-027.md) (Decision items 2 & 4, Consequences) and `INDEX.md` line 42 — verified: no other `docs/decisions/*.md` cites `DR-007`. ## Consequences

**Positive:**

- The `DR-007` register slot survives gap-free, so the sequential `max(DR-NNN)+1`
  numbering rule (project CLAUDE.md Decision Register) holds and the actual inbound
  references — [DR-027](DR-027.md) (Decision items 2 & 4, Consequences) and `INDEX.md`
  line 42 — keep resolving.
- No ScroogeLLM plugin-architecture detail (the defensible proxy-layer IP per
  [DR-027](DR-027.md) Context) remains in the public `harvest316/minspec` repo — only
  a pointer does.
- The redirect is explicit and dual-encoded — human-readable blockquote + machine-
  readable `relates_to: [DR-027]` — so tooling and readers both reach the live DR in
  `harvest316/scroogellm`.

**Negative:**

- A permanent stub with a `title:` that no longer matches its body is a small,
  standing source of confusion; the blockquote is the only thing preventing a
  misread.
- Anyone needing the actual plugin-architecture decision must have access to the
  **private** `harvest316/scroogellm` repo — public-repo readers hit a dead end by
  design.
- The decision is now split across two registers (public tombstone, private real DR
  numbered `DR-007`), creating a two-place-maintenance burden if the title or status
  ever needs updating.

## Alternatives Considered

- **Delete `DR-007` entirely.** Rejected: it would gap the `DR-NNN` sequence and break
  the inbound references that actually exist — [DR-027](DR-027.md) (Decision items 2 & 4,
  Consequences) and `INDEX.md` line 42. Tombstoning instead of deleting is exactly what
  [DR-027](DR-027.md) Decision item 4 prescribes for the moved entries ("Tombstones
  replace the moved register entries … so the sequential register stays gap-free").
- **Leave the full plugin-architecture text in place, publicly.** Rejected for the same
  reason the parent move exists — the proxy-layer logic is the moat and "cannot be
  public" ([DR-027](DR-027.md) Context). A tombstone keeps the slot without leaking the
  content.
- **Keep a private git submodule so `DR-007` stays in the public tree but points at
  private content.** Rejected at the parent level ([DR-027](DR-027.md) Alternatives) —
  a submodule leaks the name/existence and adds complexity for a solo dev; a flat
  tombstone + separate private repo is simpler.

## Costly to Refactor

Cheap to reverse — **yes, well under a day.** This stub is ~10 lines of prose and
frontmatter with no code or build dependency. Reverting (e.g. if ScroogeLLM were ever
made public again) is a copy-back of the real `DR-007` from `harvest316/scroogellm`,
gated only by the public/private decision in [DR-027](DR-027.md), which is the actual
load-bearing, costly-to-undo decision (it required a `git filter-repo` history rewrite
that changed every `main` SHA). This tombstone rides on that; it adds no blast radius of
its own.
