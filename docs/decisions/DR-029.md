# DR-029: LLM-authored self-audit appendix + earned-trust verification ("just enough human")

> **Keystone.** Supersedes the human-authors-content posture of [DR-026](DR-026.md)
> (scoped carve-out), reframes [DR-020](DR-020.md), and promotes [DR-028](DR-028.md)
> into an authoring sequence. Decided in a long design session backed by two
> multi-agent panels (constraint reconciliation + 5-lens design panel with adversarial
> critique). Security boundary split to **[DR-030](DR-030.md)**.

## Context

MinSpec's core goal: **ensure the LLM thoroughly considers all aspects of a planned
change**, with **"just enough human"** — the human writes only the trigger prompt,
reviews the core (Context / Requirements / Out-of-Scope / Open Questions), answers
OQs, raises concerns, and skims a final result. The LLM does all writing, including
the **self-audit sections** (Risks, Consequences, …) that exist to make it
cross-check its own work.

This forces an inversion of the earlier model. Tiering historically tied **ceremony**
(how much the human authors/reads) to **consideration** (how thoroughly the change is
reasoned) *only because the cost was human authoring*. Now the LLM authors, so:
**consideration should be ~thorough on every change (near-free), while ceremony stays
strictly tier-proportional.** The self-audit family becomes a *consideration
instrument* the LLM always fills deeply; tier scales only **which** sections emit and
**how much human attention** they earn — never how hard the LLM thinks.

The earlier decisions assumed the *human* must perform the risk-reasoning:
- DR-026 "Stub, not content / content always author-written / auto-fill rejected".
- DR-020 "the value is forcing the **author** to reason; 'None material' must be **written**".

The session reversed the premise: **the value is forcing the *LLM* to reason; the
human verifies a *signal*, not the content.** "A Risks section with real, per-FR
risks tells me the LLM thought carefully, so I don't have to read it."

**The crux risk this DR must answer.** Existence/count is a *gameable* proxy — a model
emits N plausible risks whether or not it reasoned (the exact slop MinSpec sells
against; the Stella-Lorenzo fabrication lands directly in the unread section). So
"the human doesn't read it" is only safe if the signal is **earned**, and the design
must be *honest* about when it is not.

## Decision

A **self-audit appendix**, **LLM-authored-last** in a new **cross-checks** phase, trusted
via a **layered, earned, tiered signal** — with the trust claim labelled honestly so
it never over-states the verification that actually ran.

### 1. Consideration-vs-ceremony — tier-scaled section family

**Breadth vs depth — only one scales (clarify).** Two things could ramp with tier;
only one does.
- **Depth** — how hard the LLM reasons about an aspect it does examine — is
  **constant**: always thorough, never tier-gated (it is near-free, so there is no
  reason to think less hard about a small change).
- **Breadth** — how many self-audit sections emit — **ramps** by tier (table below).
- **Human attention** (ceremony) ramps by tier.

**Caveat (load-bearing, R6):** tier = **size**, and size ≠ difficulty (validated,
n=120, κ=0.80). So ramping *breadth* by a size-tier under-serves the subtle small
change — the slop pocket. Two compensations: (a) seat the **cheap** sections
(Assumptions, a test-thought) as low as **T2**, so a small-but-tricky change still
gets considered; (b) offer the reality-check at **all** tiers. The principled fix is
to ramp breadth by **difficulty/reach** (DR-022 / #91) once that validates; size-tier
is the interim proxy.

| minTier | Sections (cumulative) |
|---|---|
| T1 | Risks & Mitigations — the existing DR-020 one-line floor (the **only** T1 member; zero net-new ceremony) |
| T2 | + Out-of-Scope machine cross-check · **Assumptions** (one-line) · **Test-thought** (one-line: how the change is verified) — the *cheap consideration floor*: LLM-authored, skim-not-read, so it blunts size≠difficulty (R6) without growing ceremony |
| T3 | + Consequences (± shape) · Failure-Modes/Edge-Cases · Test/Verification-Strategy (full per-FR matrix) · Alternatives-Considered |
| T4 | + Dependencies & Blast-Radius · Rollback/Reversibility |

**Per-FR disposition (anti-gaming keeper).** Coverage is not a risk *count*: every
`FR-N` carries an explicit disposition — a risk row, **or** a written
"no distinct risk — covered by FR-M", **or** "happy-path only, accepted". Harder to
fake than N floating risks (each FR must be *paired* with a reason) and a wall of
identical escapes is visible at a glance. The DR-020 written-escape is preserved
**per FR**, so coverage never becomes a tooling block.

### 2. Two-zone document = the just-enough-human line, made physical
- **Zone A "READ THIS"** (top, human-reviewed, under a new `coreHash`): One-sentence
  scope, Context, Requirements (FR-N), Out-of-Scope, Open Questions, Invariants.
  Delimited by `<!-- minspec:core-end -->`.
- **Zone B "self-audit appendix"** (below `---`, LLM-authored-last), split by
  verification status:
  - **B1 "skim"** — passed the deterministic floor (and, if it ran, the reality-check); collapsed by default.
  - **B2 "PLEASE READ"** — pulled to the top, eyes-on. A section lands in B2 if no deterministic check can vouch for it (Assumptions, Alternatives, Rollback are *structurally* always B2), it tripped specificity, the agent flagged it / could not run, or it is stale. **Degrade makes the human read *more*, never less.**
- **Dynamic heading tells the truth** about trust level (see §6).

### 3. Trust stack — earned, layered, honest about its limits
**Tier-0 deterministic floor (always on; catches *omission*, not *vacuity* — stated, not hidden):**
- **L0 per-FR disposition coverage** — names naked FRs ("FR-7: no risk, no test").
- **L1 specificity** — a self-audit line referencing no concrete anchor (FR-id / allowlist file / invariant / DR) is **flagged to B2** (not silently dropped — that would break offer-never-silent; the DROP option is **rejected**, see Alternatives).
- **L2 consistency** — **id-based** (not lexical): out-of-scope items absent from FR-set; no FR both in/out; all OQs resolved before cross-checks completes.
- **L3 freshness** — self-audit bound to a hash of **FR ids *and* FR-body bytes** (amends DR-028's ids-only rule; closes substance-rot where a reworded FR under a stable id read "fresh"). False-positive churn from cosmetic rewords is the accepted safe direction.
- **L4 tautology-aware stub check** (extends SPEC-006) — a test cell is green only if the test is non-stub **and** not assertion-free/tautological.

**Tier-1 amplifier (opt-in, degrades to the floor, never *is* the floor):**
- **L5 reality-check** — independent second reviewer (§5). **Default two-lens at T3.**
- **L6 round-table** — T3/T4 opt-in, Scrooge-metered (§5).

### 4. Two-step approval + cross-checks lifecycle (T3/T4 only)
`specify [+plan +tasks +implement] → core-signoff (human) → cross-checks (LLM) → final-approve (human) → done.`
- **core-signoff** = human declares "no further concerns on the core"; freezes the FR-set (`coreHash`, Zone-A-only soft freeze). **Not** the DR-012 gate; does not unlock src-edit.
- **cross-checks** = on core-signoff, MinSpec **void-then-offers** (never auto-on-save): the deterministic fill (matrix + skeletons) is one offer; the Tier-1 reality-check is a separate explicit offer (offer-never-silent, DR-026; never-auto-on-human-save, SPEC-010 FR-10). Editing an FR after signoff **voids** signoff and **waits** for a fresh explicit re-signoff.
- **final-approve** = the existing DR-012 full-file hash approval (`specHash`) — the **sole blocking gate**, unchanged.
- **T1/T2:** no core-signoff, no cross-checks, no second approval — T1 keeps its one Risks line. The two-step lifecycle is **T3/T4 only** (DR-012 enforcement is T3/T4).

No deterministic layer blocks final-approve (advisory only); the reality-check verdict is **pure advisory** — it floats flagged items to B2, adds **zero** friction clicks, is **not** logged or shamed ("classifier suggests, human decides", Invariant #5).

### 5. Reality-check agent + round-table (Tier-1, EPIC-007)
- Independent adversarial reviewer, a **different agent/prompt** than the author (no grading its own homework). Runs in cross-checks after the self-audit is written.
- Tier-0 gaps are passed as **exclusions, not a to-do list** ("these FRs are already covered — spend all tokens on what the matrix cannot see: cross-FR interactions, specific-but-wrong mitigations, omitted failure modes, the alternative the author skipped").
- **Default two-lens at T3** ("find the bug" + "defend the spec") to decorrelate a same-model reviewer (~2× the cheap-check cost, accepted).
- **Round-table** (T3/T4 opt-in): N lenses debate; Scrooge-metered cost shown ("round-table: $0.40, 3 concerns"). Advisory, never blocks.
- Lives in the **agent-execute** extension (Tier-1, DR-015) — **never** in `packages/minspec`/`shared` (Invariant #1). The core consumes the verdict JSON; absent → degrade to the deterministic floor with an honest label.

### 6. No trust claim until the study clears — mechanism ships now, the claim waits for evidence
The mechanism (LLM-authored self-audit + deterministic floor + two-zone doc) **ships now**; the **"skim instead of read" claim does not** — it is withheld until the anticipation-rate study clears. This is the simplest fix to the CDD "invariant-test-first" deviation (§Methodology Fit): we never ship a trust claim ahead of its test, so there is no false "validated" to label. v1 makes **no claim**, so there is nothing to over-state. (The product owner is the sole user for the first weeks; no one is misled in the interim.)
- **v1 — no claim (default until study clears):** `## Appendix — Self-Audit · read what you want`. The human reads/skims by their own judgment; the layout still sorts B1/B2 and (if the agent ran) stamps "reality-checked", but **grants no skim licence**.
- **Post-study — claim earned:** `## Appendix — Verified · skim only` activates once the study clears its bar. (Tracked: [#127](https://github.com/harvest316/minspec/issues/127).)

The **validation study** unlocks that claim. It is **passive, dogfooded, zero-extra-reading** (decided in-session — the owner is the guinea pig but will not read more than today):
- **Ground truth = this project's own bug ledger**, which RCDD ([DR-003](DR-003.md)) already produces (every fix roots-causes its bug). No pre-built escaped-defect corpus required (that was the long pole — removed).
- **One added tag per fix:** "was this risk named in the owning spec's self-audit appendix? y/n" — piggybacks on the root-cause step RCDD already performs.
- **Metric = anticipation rate:** of escaped bugs, the fraction the appendix *already foresaw*. High → the LLM's rigour is predictive → the skim claim activates. Bugs from risks the appendix never named → rigour is decorative → no claim, keep reading.
- **Needs broad rigour for data:** the LLM self-audit (*consideration*) runs broadly, decoupled from *ceremony* (which stays T3/T4), so most changes carry an appendix to measure against — without growing the human's load.
- **Pooled across users (opt-in, anonymous):** beyond the solo dogfood, anticipation-rate tuples may be contributed anonymously by consenting users to clear the global claim faster (design + consent + invariant-#1/#2 boundary tracked at [#128](https://github.com/harvest316/minspec/issues/128)).
- **Override available:** the owner may activate the claim pre-study, knowingly accepting the R3 residual.

Honest limits: passive ⇒ **slow** (waits for bugs to accumulate); attribution is fuzzy (which spec owns a bug); single-arm ⇒ measures *prediction*, not proven *causation* (no no-rigour control). Accepted as the zero-burden trade.

### 7. Reconciliation with DR-024 / Fork-B (tier → derived)
This design is tier-keyed (`minTier` per section; T3/T4-only lifecycle). [DR-024](DR-024.md)
accepted Fork B: `tier` becomes a **derived field** of a risk-profile container once
reach validates (#91). Reconciliation: the family's `minTier` keys off **whatever the
canonical tier resolves to** — scalar today, derived-from-reach later. The
section-family map and lifecycle bind to the *resolved tier value*, not to the
size-classifier specifically; if reach supersedes size as the tier source, the family
map is unchanged. No part of DR-029 depends on tier being diff-size-derived.

### 8. Build slices (vertical, sequential — not big-bang)
This DR is the **direction**; the build is sliced by *capability*, sequential, with the
study running passively throughout. **The study is global and one-time — never
per-section**; sections widen by cheap registry edits and inherit the cleared claim
(so no "one week of stats per section").
- **Slice 1 (first; no trust claim):** cross-checks phase + LLM authors the full self-audit
  family + Tier-0 deterministic floor (disposition-coverage, specificity, consistency,
  freshness, tautology-stub) + two-zone doc + B2-eyes-on. Ships value with zero study. → **SPEC-013**.
- **Slice 2:** reality-check second reviewer (Tier-1). → **new EPIC-007 spec**.
- **Slice 3:** round-table (Tier-1, T3/T4 opt-in). → same EPIC-007 spec.
- **Cross-cutting (from Slice 1):** passive anticipation-rate study; clears once →
  activates the skim claim ([#127](https://github.com/harvest316/minspec/issues/127)),
  optionally accelerated by pooled anon telemetry ([#128](https://github.com/harvest316/minspec/issues/128)).
Each slice is end-to-end and shippable (CDD vertical-slice); no slice waits on the study.

## Consequences

**Positive:**
- DR-026's "stub-not-content" is **carved out for Zone B** post-signoff (offered, never silent, dirty-editor-safe — all DR-026 postures kept); Zone A stays fully under DR-026 — so the just-enough-human reversal lands without breaking the offer-never-silent / dirty-editor invariants the rest of the suite relies on.
- DR-020's value is **reframed** (LLM reasons; per-FR disposition + specificity + agent-verdict make it machine-verifiable) rather than discarded; its tier-proportional depth table + T1 floor are kept verbatim, so existing T1 specs see zero net-new ceremony.
- DR-028 is **promoted** to the cross-checks authoring sequence; its freshness rule is **amended** to bind FR-body bytes too (L3), closing the substance-rot hole where a reworded FR under a stable id read "fresh".
- DR-012 stays the sole blocking gate; gains a `coreHash` companion (T3/T4) — the new two-step lifecycle adds a Zone-A soft freeze without introducing a second blocking gate.

**Negative:**
- The deterministic floor is **specified, not shipped** — SPEC-010 (coverage DAG), SPEC-006 (stub scanner), SPEC-013 (registry+parser) are all `specifying`. Until they implement there is **no floor**; the skim story is sequenced behind them (Follow-ups), so DR-029's headline benefit is deferred, not delivered, on acceptance.
- Adaptive rubber-stamp instrumentation is **rejected** (no per-user behavioral tracking, even local — privacy line), so the B2-eyes-on layout is the *sole* rubber-stamp nudge — the R3 residual (no tooling forces the human to read B2) is accepted as the price of no-tracking.
- The vacuity hole (R1) is unclosed by the always-on floor: a specific-AND-vacuous line clears every Tier-0 layer, and only the opt-in/default-absent reality-check (L5) catches it — so on the default deterministic-only path the appendix can be specific yet hollow.

## Alternatives considered

- **Ship the skim *claim* now on the proxy (honest-label compromise).** Considered, then **rejected** as a CDD invariant-first deviation. Adopted instead: ship the **mechanism** now, withhold the **claim** until the study clears (§6) — the panel's gate, scoped to the claim only so the mechanism is not delayed.
- **Silently drop boilerplate (L1 = delete).** Rejected: destructive auto-edit of authored content contradicts offer-never-silent (DR-026). L1 **flags to B2** instead.
- **Count-based "≥N risks" signal.** Rejected: trivially gamed; replaced by per-FR disposition + specificity + reality-check.
- **Cross-checks auto-fires on core-signoff.** Rejected: reads as auto-on-save; it is **void-then-offer** (§4).
- **Deterministic layers block approval.** Rejected: only DR-012 blocks; floor is advisory.
- **Two-step lifecycle at all tiers.** Rejected: bloats T1/T2 (the "16-criteria-for-a-bugfix" failure SPEC-001 mocks); T3/T4 only.

## Costly to Refactor

**Not undoable in <1 day — this is correctly a T4 keystone DR.** The expensive,
hard-to-reverse commitments:

- **The two-zone document format** (`<!-- minspec:core-end -->` delimiter, `coreHash`
  over Zone A, B1/B2 split). Once specs are authored in this shape and `coreHash` is
  wired into the two-step lifecycle (§4), every existing spec and the parser/registry
  (SPEC-013) encode the split; reverting to a single-zone doc means a migration of all
  spec bodies plus unwinding `coreHash` from core-signoff. Days, not hours.
- **The DR-028 freshness amendment** (L3 now binds FR-body bytes, not ids-only).
  Reverting re-opens the substance-rot hole this DR closed; and any self-audit hashes
  computed under the byte-binding rule would need recomputation. Coupled change.
- **The layered trust stack the claim sits on** (§3, §6): the L0–L4 deterministic
  floor (per-FR disposition, specificity, id-based consistency, FR-body-byte
  freshness, tautology-stub) spread across SPEC-010/006/013. The skim claim depends on
  this floor existing; tearing it out once specs author against it is the expensive
  direction (multi-spec rework, not a config edit). The heading copy itself (`v1 — no
  claim` → `Verified · skim only`, gated on [#127](https://github.com/harvest316/minspec/issues/127))
  is the cheap half — a one-line flip, correctly listed under *Cheap to reverse* below.
- **Cheap to reverse (the ADR-filter "yes" parts):** the per-section `minTier` registry
  map (§1 table) is "cheap registry edits" by the DR's own §8; widening/narrowing which
  tier emits which section, and toggling the reality-check default (L5), are
  config-level and same-day. The DR-024/Fork-B reconciliation (§7) was deliberately
  written so the family map binds to the *resolved tier value*, not the size-classifier
  — so a later switch to reach-derived tiers (#91) costs nothing here.

ADR-filter answer: **No — cannot be undone in <1 day** for the format/lifecycle/floor
commitments, which is why it is a DR rather than a just-do-it; the claim-flip and the
registry map are the same-day-reversible knobs deliberately isolated from the keystone.
